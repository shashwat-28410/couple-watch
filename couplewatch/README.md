# ♡ CoupleWatch

A premium, romantic, and modern real-time co-watching platform designed for couples to watch videos and connect in real-time. Built with a sleek HSL-accented dark cinematic layout, it enables synchronized media playback, integrated video calls with draggable overlays, live room chats, and privacy-first watch history tracking.

---

## ✨ Features

### 🎬 1. Synchronized Video Player
*   **HMR Synchronized Playback:** Real-time pause, play, and timeline seeking fully synced across the room using Supabase Realtime.
*   **Independent Local View Modes:** Both the **Host** and the **Partner** have independent local controls for `Fit` (letterbox), `Fill` (zoom/stretch), and custom `Zoom` levels (up to 150%) with pan-and-drag. Adjusting your zoom will never interrupt your partner's viewing experience.

### 📹 2. Fullscreen Video Call Overlay
*   **WebRTC Video Call:** Integrated low-latency voice and video call with your partner.
*   **Draggable Overlay:** When entering fullscreen mode on the player, your partner's camera turns into a floating PIP box.
*   **Fluid Repositioning:** Click and drag the camera overlay to any corner of the screen so it never blocks key parts of the video.
*   **Minimize Toggle:** Collapse the remote feed into a subtle, pulsating camera icon for maximum movie immersion, and expand it with a single tap.

### 🛡️ 3. Privacy-First Watch History
*   **History Opt-in:** Watch History tracking is **OFF by default** to protect user privacy. A global toggle in the History tab enables/disables it (saved securely to `localStorage`).
*   **Premium Visual Stats:** High-quality HSL-styled SVG statistics tracking overall watch time and sessions.
*   **Intelligent Continue Watching:** Tracks incomplete videos with interactive "Continue Watching" cards that start a synced room right where you left off.
*   **Granular Controls:** Delete individual cards via interactive trash buttons or wipe the entire history using "Clear All," both backed by double-confirmation modals.

### 💬 4. Interactive Live Chat
*   **Real-time Messaging:** Chat directly next to the player with elegant bubbles, timestamps, and typing feedback.
*   **Quick Reactions:** Float transparent emoji reactions across the video stream in real-time.

## 🛠️ Technology Stack
*   **Frontend:** React 19, Vite, Tailwind CSS, React Router v7
*   **Backend & Database:** Supabase (Database, Auth, and Realtime Listeners)
*   **Real-Time Media:** WebRTC (Peer-to-peer audio/video streaming)
*   **Media Player:** `react-player` for standard/custom HTML5 and streaming URL playback

## 💾 Supabase Database Schema
To set up your Supabase project, execute the following SQL DDL commands in your Supabase SQL Editor:

```sql
-- 1. Profiles Table (Linked with Supabase Auth users)
create table public.profiles (
  id uuid references auth.users on delete cascade primary key,
  full_name text,
  avatar text,
  updated_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 2. Rooms Table
create table public.rooms (
  id bigint generated by default as identity primary key,
  room_code text unique not null,
  created_by uuid references auth.users on delete cascade not null,
  host_id uuid references auth.users on delete set null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 3. Room Members Table
create table public.room_members (
  id bigint generated by default as identity primary key,
  room_id bigint references public.rooms on delete cascade not null,
  user_id uuid references auth.users on delete cascade not null,
  role text not null default 'member', -- 'host' or 'member'
  joined_at timestamp with time zone default timezone('utc'::text, now()) not null,
  unique(room_id, user_id)
);

-- 4. Room State Table (Handles synced playback state)
create table public.room_state (
  id bigint generated by default as identity primary key,
  room_id bigint references public.rooms on delete cascade not null unique,
  is_playing boolean default false not null,
  current_timestamp_seconds double precision default 0.0 not null,
  video_url text,
  updated_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 5. Room Chat Messages Table
create table public.messages (
  id bigint generated by default as identity primary key,
  room_id bigint references public.rooms on delete cascade not null,
  user_id uuid references auth.users on delete cascade not null,
  content text not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 6. Watch History Table (Privacy-controlled tracking)
create table public.watch_history (
  id uuid default gen_random_uuid() primary key,
  video_title text,
  video_url text not null,
  participants uuid[] not null,
  started_at timestamp with time zone default timezone('utc'::text, now()) not null,
  ended_at timestamp with time zone,
  last_position_seconds double precision default 0.0 not null,
  total_watched_seconds double precision default 0.0 not null
);

-- 7. Automated profile creation trigger
create or replace function public.handle_new_user()
returns trigger as $$
begin
  insert into public.profiles (id, full_name, avatar)
  values (new.id, split_part(new.email, '@', 1), null);
  return new;
end;
$$ language plpgsql security definer;

create trigger on_auth_user_created
  after insert on auth.users
  for each row execute procedure public.handle_new_user();
```

## 🚀 Installation & Running

1. **Clone the Repository:**
   ```bash
   cd couple-watch/couplewatch
   ```

2. **Install Dependencies:**
   ```bash
   npm install
   ```

3. **Start Development Server:**
   ```bash
   npm run dev
   ```
   The site will load locally on [http://localhost:5173](http://localhost:5173) (or the port displayed in your terminal).

4. **Production Build:**
   ```bash
   npm run build
   ```
